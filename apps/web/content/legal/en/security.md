# Security Policy – Proovra

Last Updated: 08.02.2026

Proovra is built for trust-sensitive workflows. This Security Policy describes our security approach and how to report vulnerabilities.

## 1. SECURITY PRINCIPLES

We aim to protect:

- confidentiality of account data,
- integrity of evidence artifacts (hashes/signatures/custody logs),
- availability and reliability of Services.

## 2. TECHNICAL AND ORGANIZATIONAL MEASURES

Measures may include:

- encryption in transit (TLS) and at rest where applicable,
- access controls and least-privilege permissions,
- audit logs for sensitive operations,
- separation of environments (development vs production),
- secure key management for signing keys,
- monitoring for abuse, anomaly detection, and rate limiting,
- routine dependency updates and patch management.

## 3. EVIDENCE INTEGRITY MODEL

Proovra provides cryptographic integrity verification:

- hashes detect alterations to stored evidence,
- signatures attest to integrity artifacts generated by the system at completion time,
- custody timeline events are designed to be auditable.

Important: integrity verification is not the same as proving who created content, where it came from, or whether it is truthful.

## 4. CUSTOMER RESPONSIBILITIES

You are responsible for:

- protecting your credentials and devices,
- ensuring proper authorization within your team,
- preventing unauthorized sharing of verification links,
- using strong passwords and secure authentication methods.

## 5. VULNERABILITY DISCLOSURE

If you believe you found a vulnerability, please report it responsibly:

security@proovra.com

Include:

- description and impact,
- reproduction steps,
- affected URLs/components,
- proof-of-concept (if safe).

We request you:

- do not exploit beyond what is necessary,
- do not access other users’ data,
- do not disclose publicly before we investigate.

## 6. SECURITY INCIDENTS

In case of a security incident affecting user data, we will evaluate obligations under applicable law and notify impacted users and authorities where required.
